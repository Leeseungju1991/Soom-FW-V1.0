# ICONIA Firmware + Cloud Server

ICONIA 프로젝트용 펌웨어와 서버 코드입니다.

---

## 주요 기능

- BLE를 이용한 Wi-Fi 프로비저닝
- 터치 입력으로 Deep Sleep 해제
- 배터리 상태 측정
- 카메라 JPEG 촬영
- Wi-Fi HTTPS 업로드
- 서버 명령으로 재프로비저닝 진입

---

## 전체 동작

### 1. 최초 설정

처음 부팅했을 때 Wi-Fi 정보가 없으면,
디바이스는 BLE 프로비저닝 모드로 들어갑니다.

앱이 BLE로 다음 정보를 전달합니다.

- Wi-Fi SSID
- Wi-Fi Password

펌웨어는 받은 정보로 Wi-Fi 연결을 시도하고,
성공하면 내부 저장소(NVS)에 저장합니다.

이후부터는 다시 부팅해도 매번 프로비저닝 모드로 들어가지 않습니다.

---

### 2. 평상시 동작

평소에는 Deep Sleep 상태로 대기합니다.

터치가 발생하면 디바이스가 깨어나서 다음 순서로 동작합니다.

1. 어떤 방향의 터치인지 확인
2. 배터리 상태 측정
3. 카메라로 JPEG 이미지 촬영
4. Wi-Fi 연결
5. 서버로 이벤트 업로드
6. 완료 후 다시 Deep Sleep 복귀

즉, 항상 켜져 있는 구조가 아니라,
이벤트가 발생했을 때만 잠깐 활성화되는 방식입니다.

---

### 3. 서버 업로드

펌웨어는 서버로 아래 데이터를 보냅니다.

- 터치 방향
- 디바이스 ID
- 배터리 잔량
- 촬영 이미지

업로드는 `multipart/form-data` 형식의 HTTPS 요청으로 보냅니다.

서버는 이 데이터를 받아서:

- 이미지 저장
- 디바이스 식별
- 상태 기록
- 다음 동작 명령 결정

을 처리합니다.

---

### 4. 재프로비저닝

일반 동작 중에는 BLE 프로비저닝 모드로 들어가지 않습니다.

다만 서버가 특정 디바이스에 대해
`enter_provisioning` 명령을 응답으로 내려주면,
펌웨어는 저장된 Wi-Fi 정보를 삭제하고
즉시 BLE 프로비저닝 모드로 다시 들어갑니다.

이 기능은 다음 상황에서 사용할 수 있습니다.

- Wi-Fi AP가 변경된 경우
- 설치 장소 네트워크가 바뀐 경우
- 원격으로 네트워크 재설정을 시키고 싶은 경우

---

## 펌웨어 역할

펌웨어는 디바이스 안에서 아래 기능만 담당합니다.

- BLE 프로비저닝
- Wi-Fi 연결
- 터치 감지 후 wake-up 처리
- 배터리 측정
- 카메라 촬영
- 서버 업로드
- 서버 명령 해석
- Deep Sleep 복귀

즉, 펌웨어는 센서/카메라/네트워크 업로드까지 담당하고,
저장이나 후처리 같은 클라우드 로직은 서버가 담당합니다.

---

## 서버 역할

서버는 아래 기능을 담당합니다.

- 펌웨어 업로드 요청 수신
- API Key 검증
- 이미지 저장
- 디바이스별 명령 관리
- 재프로비저닝 명령 전달

즉, 서버는 펌웨어가 올린 이벤트를 받고,
필요한 저장과 제어 결정을 하는 역할입니다.

---

## 메모리 최적화 방향

펌웨어는 ESP32 환경을 고려해서
메모리를 많이 쓰지 않도록 구성했습니다.

적용한 방향은 다음과 같습니다.

- 큰 문자열 조립 최소화
- multipart 데이터 순차 전송
- 고정 크기 버퍼 사용
- 카메라/와이파이 필요 시에만 활성화
- 업로드 후 즉시 자원 정리

즉, 이벤트 처리 순간에만 필요한 기능을 켰다가
끝나면 바로 끄는 방식입니다.
